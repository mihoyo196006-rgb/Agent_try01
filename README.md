# Morning Feishu Agent

This repo supports the DayFlow cloud reminder path.

## Current Path

Daily delivery now runs entirely on Cloudflare Worker/KV:

1. Dayflow desktop saves state.
2. `D:\try\cloud-sync.js` posts a compact snapshot to the Worker.
3. Cloudflare Cron invokes the Worker at 09:00, 09:10, and 09:20 Beijing time.
4. The Worker reads yesterday's snapshot, sends one Feishu bot message, and records a KV delivery marker.

The previous Windows scheduled local snapshot path and GitHub Actions scheduled sender path have been removed from the active daily route.

## Token And Quota Use

The default message is generated by deterministic templates. It does not call an LLM and spends 0 model tokens.

Cloudflare Cron may invoke the Worker up to three times each morning. KV delivery markers and Feishu message UUIDs make the sends idempotent: after the first successful send, later invocations skip. Dayflow desktop writes are debounced and unchanged snapshots are skipped.

## Required Cloudflare Secrets

- `DAYFLOW_WRITE_TOKEN`
- `DAYFLOW_READ_TOKEN`
- `FEISHU_APP_SECRET`

## Required Cloudflare Vars

- `FEISHU_APP_ID`
- `FEISHU_USER_ID`

`wrangler.toml` configures the Worker cron triggers and non-secret vars. Do not commit Feishu app secrets or DayFlow tokens.
